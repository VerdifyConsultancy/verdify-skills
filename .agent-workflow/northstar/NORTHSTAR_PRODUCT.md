## PRODUCT-001 Purpose And Outcome

- Product purpose: Verdify Skills is the reusable lifecycle, planning, and learning-capture contract layer for Jason's self-hosted software-and-knowledge operating plane: source control, GitHub Issues/PRs, CI/CD, pipelines, k3s management, agent orchestration, evidence ingestion, North Star planning, and Gravity-style local evidence workflows.
- Current state: The repo has lifecycle skills, transcript intake, research ingestion, draft North Star synthesis, readiness-loop skills, validation, tests, and route logic.
- Target state: Every downstream project can feed context into a self-improving North Star loop, receive evidence-backed product and architecture drafts, iterate through research and feedback, capture validated lessons as proposed improvements, and use locked North Star artifacts as the root planning authority for definition, architecture, platform readiness, CI/CD wave delivery, review, release, and reusable Gravity evidence-ingestion work. External OpenClaw workflow skills expose that lifecycle as a small user-facing set of ideation-to-production workflows while GitHub and `.agent-workflow` remain authoritative. Repo-controller bootstrap lets each active project get organized through standard self-discovery, repo-agent scope/responsibility charters, fleet observability, namespace/storage/credential mapping, and issue/PR-backed gap closure before broad autonomous rollout.
- Non-goals: Do not start Gravity feature implementation, grant production mutation rights, or replace GitHub as delivery control plane from this artifact loop.
- Success measures: Active issues, waves, PRs, tests, deployments, final approvals, and ADRs can cite stable North Star IDs and evidence references.
- Evidence: all registered evidence items, especially `northstar://evidence/NSE-20260623-agent-platform-live-state-audit`, `northstar://evidence/NSE-20260623-gravity-remote-onyx-confirmation`, `northstar://evidence/NSE-20260623-sunshine-gravity-extraction-architecture-audit`, `northstar://evidence/NSE-20260623-openclaw-hermes-reuse-interface-security-audit`, `northstar://evidence/NSE-20260623-secrets-credential-injection-patterns`, `northstar://evidence/NSE-20260623-review-inbox-skill-implementation-best-p`, `northstar://evidence/NSE-20260623-wave-release-planning-implementation-bes`, `northstar://evidence/NSE-20260623-observability-diagnostics-implementation`, `northstar://evidence/NSE-20260623-session-ledger-implementation-best-pract`, `northstar://evidence/NSE-20260623-agent-platform-control-implementation-be`, `northstar://evidence/NSE-20260623-github-backlog-sync-implementation-best`, `northstar://evidence/NSE-20260623-environment-gitops-implementation-best-p`, `northstar://evidence/NSE-20260623-gravity-core-extraction-implementation-b`, and `northstar://evidence/NSE-20260623-learning-capture-implementation-best-pra` for the latest research loop.

## PRODUCT-003 PRD Summary

- Problem: Agentic delivery can drift, hide decisions in chat, overreach permissions, and ask humans to review incomplete work unless product intent, architecture intent, evidence, planning iteration, and final approvals are durable.
- Users: Jason, James, project owners, worktree agents, reviewers, controller agents, platform operators.
- Jobs to be done: ingest evidence, synthesize North Star product/architecture artifacts, answer or route planning questions, propose research, iterate design, route lifecycle work, execute bounded lanes, deploy reviewable previews, expose OpenClaw-managed SDLC workflow skills, and preserve audit history.
- Minimum useful scope: research ingest, transcript replan, North Star artifact loop, repo hygiene, controller-loop specification, platform readiness, Gravity readiness, state-of-union, sprint planning, lane execution, critic review, and release verification.
- Differentiators: explicit North Star governance, MCP/API-first control-plane contracts, OpenClaw-facing workflow skills, repo-controller bootstrap/self-discovery, CI/CD wave lifecycle, review inbox, session ledger, proposal-only learning capture, loop-readiness checks, fleet observability and recovery, Orbit-style operating briefs, self-hosted source-control/k3s operations, reusable Gravity evidence-ingestion engine, and traceable human-governed autonomous delivery.
- Out of scope: production autonomy before platform readiness and human approval.
- Dependencies: GitHub, repo-local instructions, `.agent-workflow` artifacts, CI/CD checks, GitOps environment reconciliation, observability, and credentials/RBAC boundaries.
- Planning loop: distill evidence, propose research, answer questions where evidence supports an answer, propose design, accept human feedback, return to the loop, and request final lock approval only when the North Star is ready to advance to the next milestone.
- Owned-IP bias: own the lifecycle/evidence/control-plane/Gravity-engine contracts and compose commodity Git, Kubernetes, GitOps, database, vector, and observability primitives rather than rebuilding them.
- Delivery model: work moves through versioned wave delivery envelopes on a deterministic controller state machine; lanes are per-wave write-conflict partitions; tasks (issues) are the smallest committed unit with their own PR and fresh-context review; planning is rolling-wave; orchestration is event-driven through a vendor-neutral worker adapter selecting Claude or Codex per host. See `ARCH-021` and `ADR-0011` through `ADR-0018`.
- Evidence: all registered evidence items.
