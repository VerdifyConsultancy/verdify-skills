# Discovery mode

Use two passes.

## Intake pass

Create source records with location, type, date if known, summary, credibility, and the claims they support. Search the repository before asking the human. Separate direct evidence from reported context and inference.

Inventory ecosystem evidence, not only product notes: stakeholders, ownership, relationships, data, integrations, infrastructure, hosting, environments, deployment, support, operations, compliance, and source-of-truth documents.

## Gap interview pass

Ask the smallest set of questions that closes material gaps. Prefer contrastive questions that expose a decision, for example: “Should the skill package be fetched ephemerally, vendored, or installed as a dependency? The current sources support all three.”

Record contradictions without selecting a side. A human answer becomes a proposed decision until approved. If a missing answer can change architecture, delivery, operations, data handling, security, compliance, cost, or public behavior, record it as a lifecycle gap rather than burying it in notes.
