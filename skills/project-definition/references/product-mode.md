# Product mode

Prioritize the smallest coherent outcome that creates value. Separate:

- must-have workflows;
- deferred workflows;
- explicit non-goals;
- assumptions that need validation;
- success metrics that can be observed after release.

A metric must define what is measured, how it is measured, and the evaluation window or threshold when applicable. Technical completion alone is not a product outcome.

Include non-feature product concerns that affect the development lifecycle: rollout, onboarding, support, operator experience, documentation, admin ownership, pricing/cost limits, legal/compliance promises, and acceptance authority. If one of these is intentionally out of scope, record it as a non-goal or lifecycle coverage rationale.
