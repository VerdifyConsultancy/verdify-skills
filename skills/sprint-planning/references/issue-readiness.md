# Issue readiness

An implementation issue should state:

- problem and desired outcome;
- evidence/context;
- observable acceptance intent;
- non-goals;
- affected users/modules when known;
- risk and deployment impact;
- parent/sub-issue and blocking relationships.
- milestone or Project placement when the sprint plan will claim timeline;
- whether the work needs QA, human review, preview deployment, telemetry,
  rollback, or release verification evidence.

The issue is not an implementation script. The lane contract may add bounded technical detail after architecture and planning review.
