# Evidence Registry Rules

## Evidence IDs

Use stable IDs generated by the CLI:

```text
NSE-YYYYMMDD-<slug>
```

Reference evidence as:

```text
northstar://evidence/<id>
```

## Required metadata

Every item needs:

- title;
- evidence type;
- evidence status;
- source path or URI;
- SHA-256 of the ingested file;
- copied source path;
- summary;
- tags;
- claims;
- limitations.

## Claims

Claims must be source-backed and phrased as evidence, not decisions. Use
planning synthesis to turn claims into requirements, stories, principles,
issues, or gates.

## Tags

Prefer short lowercase tags such as:

- `platform`
- `gravity`
- `skills`
- `observability`
- `governance`
- `security`
- `research`
- `adversarial-review`

## Query behavior

Registry queries should match ID, title, summary, tags, source URI, source path,
claims, and planning relevance. Query results should include the reference URI
and normalized evidence item path.
