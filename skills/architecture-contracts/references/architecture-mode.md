# Architecture mode

Cover only decisions necessary to support approved product behavior and quality attributes.

Minimum review areas:

- system context and actors;
- component responsibilities;
- synchronous/asynchronous communication;
- data ownership, lifecycle, retention, and migration;
- authentication, authorization, trust boundaries, and secrets;
- deployment topology, scaling, resilience, and rollback;
- observability and operational ownership;
- external dependencies and failure isolation;
- known risks and unresolved decisions.

Prefer a simple architecture that preserves future options over speculative decomposition.
